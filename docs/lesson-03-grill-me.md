# 第三课：用 grill-me 交互式理清思路并写出文档

本课目标：从产品经理日常工作出发，学会用 `/grill-me` 把“模糊需求”快速打磨成“可评审、可推进、可协作”的文档。

## 你将学会

- 用 PM 视角描述问题：业务目标、用户价值、成功指标；
- 通过连续提问澄清：范围、优先级、上线节奏、协作依赖；
- 输出可直接进入评审会的文档（PRD/方案稿/汇报稿）。

## 典型学习场景（产品经理高频）

### 场景 A：老板一句话需求

> “把转化率拉上去，下周给我方案。”

你可以这样开场：

```text
/grill-me
请以产品经理视角连续提问我，帮我把“提升注册转化率”的一句话需求，整理成下周可评审方案。
```

重点要被问清：

- 提升哪一段转化（访问->注册，还是注册->激活）；
- 当前基线和目标值；
- 能改的页面/流程边界；
- 技术、设计、投放的协作依赖。

---

### 场景 B：需求评审前，自己也没想透

> “感觉要做，但不知道先做什么，怕评审会上被问倒。”

开场话术：

```text
/grill-me
请模拟评审会，从目标、用户、边界、风险四个维度连续问我，直到可以形成评审文档。
```

重点要被问清：

- 这是“必须做”还是“值得做”；
- 主目标用户是谁，不做谁；
- MVP 最小范围是什么；
- 失败风险和回滚方案。

---

### 场景 C：两个方案纠结，无法取舍

> “做聊天式引导，还是表单式引导？”

开场话术：

```text
/grill-me
我有 A/B 两种方案，请你逐项追问并逼我给出取舍依据，最后输出决策建议。
```

重点要被问清：

- 比较维度（开发成本、上线时间、用户学习成本、数据可观测性）；
- 哪些假设需要实验验证；
- 决策门槛（达到什么指标就选哪条路）。

## 标准操作流程（PM 版）

## 第一步：先写 6 行“问题卡片”

```text
1) 业务目标：
2) 目标用户：
3) 当前问题：
4) 期望结果（指标）：
5) 时间边界：
6) 当前不确定点：
```

## 第二步：启动 grill-me

```text
/grill-me
请按“目标->用户->范围->方案->风险->里程碑”的顺序提问我，每次一个问题。
```

## 第三步：用“单题模板”回答

```text
- 当前判断：
- 依据/数据：
- 约束条件：
- 待验证假设：
```

如果问题过大，直接说：

```text
请把这个问题拆成两个更具体的问题，并给我可选项。
```

## 第四步：收敛成评审文档

```text
请把我们问答整理成评审稿，结构为：
1) 背景与业务目标
2) 用户与场景
3) 需求范围（做/不做）
4) 方案对比与取舍
5) 风险与验证计划
6) 里程碑与协作分工
```

可选：进一步生成 PRD

```text
/to-prd
```

## 示例：完整学习演练（可直接照抄）

```text
我想优化新用户注册流程。
现状：注册转化率 32%，目标下月到 40%。
问题：卡在手机验证码步骤，流失高。
限制：两周内上线，研发 2 人，不能改后端账号体系。
不确定：是改文案和流程，还是增加免验证码登录。
```

然后输入：

```text
/grill-me
请按评审会方式连续提问我，直到形成一份可执行方案。
```

## 常见坑与改法（PM 常见）

- 坑 1：只说“想做什么”，不说“为什么现在做”
  - 改法：每个需求都补上业务目标与时间窗口。
- 坑 2：只讲功能，不讲指标
  - 改法：至少定义 1 个主指标 + 1 个护栏指标。
- 坑 3：讨论很久，没有结论
  - 改法：每 5 轮要求 Pi 产出一次“当前结论+待确认项”。

## 课后练习（场景化）

请从下面任选 1 个主题完成：

1. 拉新：提升落地页到注册转化率；
2. 留存：提升新用户次日留存；
3. 商业化：提高试用到付费转化率。

每个主题要求：

1. 用 `/grill-me` 跑完至少 12 轮问答；
2. 产出一份评审文档；
3. 文档中必须包含：目标指标、MVP 范围、风险与里程碑；
4. 保存到仓库并发起 commit。

完成标准：

- 团队成员不需要你口头补充，也能理解并执行；
- 技术/设计能据此拆任务；
- 评审会中能回答“为什么做、先做什么、怎么验证”。
