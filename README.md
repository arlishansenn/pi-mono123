# Pi 员工入门培训

这份文档用于员工自我训练 [Pi](https://pi.dev/)：安装 Pi、安装工具包、安装产品/PM 常用 Skills，完成第一个 `prd.md` 文件，并学会打开聊天历史。

## 为什么要培训大家使用 pi-mono

我们正在进入一个**人机共生加速发展**的时代。
要高效地与 AI 协作，就要掌握机器最擅长理解和执行的表达方式——**命令行与文本（CLI）**，而不只是图形界面（GUI）。

因此，我们希望不论是程序员还是非程序员，都能逐步建立同一种协作语言：**CLI + 自然语言指令**，并尽量使用同一套工具链。
在现阶段，`pi-mono` 因为**上手门槛低、协作链路完整**，被选为团队统一入口。

### 我个人的实践经验

1. **每个项目都有一个“思考区 repo/仓库”**
   为了让大家借助 coding agent 快速理解项目，我会把思考区仓库以只读方式开放。
   你可以直接让 AI 帮你建立项目直觉，例如：
   - “阅读本 repo/仓库，告诉我这个项目的概念模型，先帮我建立感性认识。”

2. **我有一个长期运行的个人 workspace（OpenClaw/Hermes 共享）**
   这个 workspace 是我用 `pi-mono` 维护的底层酝酿区/思考工作区。
   当想法成熟后，我会从这里分裂出单个项目的思考区，再分享给项目参与者。

3. **项目协作再进入正式开发区**
   团队在共享理解基础上，再共同进入 GitLab 开发仓库推进实现。

## 我们的工作约定（新版）

在今天，**repo/仓库 不再只是程序员开发专用区域**，而是一个**人机共生的工作区域**。

这是我们的工作约定，也建议大家采用同样方式：
**先在思考区沉淀认知，再进入开发区协作交付。**

在这种新的工作模式下，任何项目都可以被机器的力量10倍加速，这会让人有种眩晕感，请系好安全带，坐稳了，**have fun** 🚀

## 培训目标

完成培训后，员工应能：

1. 在本机安装并启动 Pi。
2. 完成第一次 `/login`，并为中国大陆员工优先选择 DeepSeek。
3. 学会指定 DeepSeek 模型，例如启动时使用 `pi --model deepseek/deepseek-v4-flash`。
4. 知道如何安装 Pi package。
5. 安装并使用 Matt Pocock 的常用 Skills。
6. 用自然语言让 Pi 生成第一个产品 PRD。
7. 用自然语言让 Pi 把文档 commit 并 push 到 GitHub。
8. 第二课学会打开 Pi 聊天历史并恢复旧会话。

## 课程正文

### 第一课：安装 Pi、设置模型、写第一个 PRD

学习内容：

- 安装并启动 Pi；
- 第一次 `/login`；
- 面向中国大陆员工选择 DeepSeek；
- 设置 DeepSeek 模型，默认使用 `deepseek/deepseek-v4-flash`；
- 安装 Matt Pocock skills；
- 用 `/grill-me` 澄清需求；
- 用 `/to-prd` 生成 `prd.md`；
- 让 Pi 自己 commit 并 push 到 GitHub。

课程文档：

- [第一课：安装 Pi、设置模型、写第一个 PRD](docs/lesson-01-first-prd.md)

### 第二课：文档写到一半，发现走到死胡同，回到中间一句提示词重新分叉，不用大段提示词重来一次

学习内容：

- 在写方案、PRD、汇报稿时低成本试不同版本；
- 让输入行保持空白；
- 连续按两次 `ESC`；
- 回到中间某一句继续分叉。
